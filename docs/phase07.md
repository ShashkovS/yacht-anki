# Фаза 07 — Статистика и настройки

## Цель
Реализовать страницу статистики обучения и страницу настроек пользователя. Дать пользователю понимание прогресса и контроль над параметрами повторения.

## Результат
Пользователь видит свою статистику обучения и может настроить параметры FSRS.

---

## 1. Страница статистики (`StatsPage`)

### Backend: агрегации
Добавить эндпоинт `POST /stats/get` (или расширить `/review/summary`).

Данные для агрегации из `review_log` и `card_states`:

### Метрики
- **Повторений за сегодня**: количество записей в review_log за текущий день
- **График по дням**: количество повторений за последние 30 дней (для столбчатой диаграммы)
- **Распределение по рейтингам**: доля Again/Hard/Good/Easy за период
- **Прогресс по колодам**: для каждой колоды — сколько карточек new/learning/review
- **Самые сложные карточки**: топ-10 по частоте Again (помогает понять, что требует внимания)
- **Streak**: сколько дней подряд были повторения
- **Общий прогресс**: процент карточек, дошедших до стадии review

### UI
- Секция "Сегодня": число повторений, средний рейтинг
- Столбчатая диаграмма повторений за 30 дней (можно средствами Tailwind CSS или простой SVG/Canvas)
- Прогресс-бары по колодам
- Таблица сложных карточек с возможностью перейти к повторению

## 2. Страница настроек (`SettingsPage`)

### Параметры
- **Целевой retention** (desired_retention): слайдер 0.70–0.97, по умолчанию 0.90
  - Пояснение: "Чем выше — тем чаще повторения, но лучше запоминание"
- **Лимит новых карточек в день**: числовое поле, по умолчанию 10
- **Лимит повторений в день**: числовое поле, по умолчанию без ограничения
  - Пояснение: "Оставьте пустым, чтобы повторять все карточки, которые пора"

### Логика
- При загрузке: `POST /settings/get`
- При сохранении: `POST /settings/save`
- Настройки применяются при следующем формировании очереди
- FSRS-планировщик на фронте пересоздаётся с новым retention
