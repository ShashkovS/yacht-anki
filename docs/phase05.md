# Фаза 05 — UI повторения и навигация

## Цель
Собрать воедино FSRS-логику (фаза 03) и движок визуализации (фаза 04) в рабочий UI для повторения карточек. Создать основные страницы приложения.

## Результат
Пользователь может залогиниться, увидеть дашборд с количеством карточек к повторению, начать сессию, отвечать на вопросы с диаграммами и получать расписание FSRS.

---

## 1. Структура страниц

### Навигация (замена шаблонной)
- **Главная / Дашборд** (`/`) — сводка: карточек к повторению, новых, streak
- **Повторение** (`/review`) — основной экран сессии
- **Колоды** (`/decks`) — список колод с прогрессом
- **Колода** (`/decks/:id`) — карточки конкретной колоды, статус каждой
- **Статистика** (`/stats`) — заглушка, реализация в фазе 07
- **Настройки** (`/settings`) — заглушка, реализация в фазе 07
- **Логин** (`/login`) — существующая страница, перевести на русский

### Роутинг
- Дашборд, повторение, колоды, статистика, настройки — требуют авторизации
- Логин — только для неавторизованных

## 2. Страница повторения (`ReviewPage`)

### Поток
1. Загрузка очереди с бэкенда (`/review/queue`)
2. Показ карточки:
   - Верх: диаграмма (DiagramStage)
   - Середина: текст вопроса
   - Низ: кнопка "Показать ответ"
3. После нажатия "Показать ответ":
   - Диаграмма обновляется (если есть "правильное" состояние)
   - Показывается текст ответа и объяснение
   - Четыре кнопки рейтинга: Не помню / Сложно / Хорошо / Легко
   - Под каждой кнопкой — следующий интервал ("через 10 мин", "через 1 день")
4. После рейтинга → следующая карточка
5. Когда очередь пуста → "Всё на сегодня!"

### Интерактивность по типам карточек
- **term_definition**: показать диаграмму + вопрос, ответ текстом
- **directional**: пользователь вращает яхту (drag handle), потом проверяется угол
- **trim**: выбор из вариантов ("выбрать шкот" / "потравить")
- **manoeuvre**: последовательность шагов, показать по порядку
- **right_of_way**: тап по яхте, которая должна уступить

## 3. Страница дашборда (`DashboardPage`)

- Приветствие с именем пользователя
- Карточка "К повторению сегодня: N" с кнопкой "Начать"
- Карточка "Новых: N"
- Мини-прогресс по колодам (процент изученных карточек)

---

## 4. Страница колод (`DecksPage`)

- Список всех встроенных колод
- Для каждой: название, описание, количество карточек, прогресс пользователя
- По клику → страница колоды

### Страница колоды (`DeckDetailPage`)
- Список карточек с их статусом (новая / на изучении / изучена)
- Кнопка "Учить эту колоду" (начать повторение только карточек из этой колоды)

---

## 5. Мобильная адаптация

- Layout под мобильный portrait mode (основной сценарий использования)
- Кнопки рейтинга — крупные, удобные для большого пальца
- Диаграмма занимает верхнюю половину экрана
- Touch-события для интерактивных карточек (вращение, тап)
