# Фаза 06 — Контент: карточки и колоды

## Цель
Создать реальный обучающий контент: ~65 карточек в 3 колодах, основанных на исследованиях из `docs/v1.md` и `docs/v2.md`. Загрузка через сиды.

## Результат
При запуске dev-сервера в БД загружаются 3 колоды с карточками. Каждая карточка имеет корректный DiagramSpec, текст вопроса/ответа и теги.

---

## 1. Колоды

### Колода 1: Терминология (~30 карточек)
Тип карточек: `term_definition`, `directional`

Примеры карточек:
- **Курсы**: бейдевинд, галфвинд, бакштаг, фордевинд, левентик — для каждого диаграмма с яхтой под нужным углом к ветру и вопрос "какой это курс?"
- **Направления**: приводиться/уваливаться — диаграмма + "куда повернёт нос?"
- **Галсы**: левый/правый галс — по положению гика определить галс
- **Ветер**: истинный/вымпельный — треугольник ветра
- **Стороны**: наветренный/подветренный — два объекта и ветер
- **Паруса и органы управления**: грот-шкот, каретка, каннингхэм, аутхаул, ахтерштаг, генакер, бушприт — термин → что это и зачем
- **Трим**: выбрать/потравить, заполаскивает, колдунчики, пузо, твист

### Колода 2: Манёвры (~15 карточек)
Тип карточек: `manoeuvre`, `trim`

Примеры карточек:
- **Приводиться**: что делать с гротом? (выбрать шкот)
- **Уваливаться**: что делать с гротом? (потравить шкот + оттяжка гика)
- **Оверштаг**: последовательность действий, что происходит с парусами
- **Фордевинд (поворот)**: последовательность, опасность переброса гика
- **Постановка генакера**: когда ставить, что делать
- **Уборка генакера**: когда убирать, последовательность
- **Работа с кареткой**: когда использовать каретку вместо шкота
- **Fine tune**: зачем нужен, когда использовать
- **Рифление**: когда и как

### Колода 3: Правила расхождения (~20 карточек)
Тип карточек: `right_of_way`

Примеры карточек:
- **Правило 10**: противоположные галсы — 3-4 варианта расположения двух яхт
- **Правило 11**: один галс, связаны — наветренная уступает подветренной
- **Правило 12**: один галс, не связаны — чисто позади уступает
- **Правило 13**: во время оверштага — теряешь все права
- **Правило 14**: избежание контакта
- **Правило 16**: изменение курса яхтой с правом дороги
- **Правило 18**: место у знака — сцены с буем и зоной
- **Связанность**: определить, связаны ли яхты по схеме
- **Чисто впереди/позади**: определить по схеме

## 2. Формат сидов

### Файл `backend/db/seed_decks.py`
- Идемпотентная загрузка: проверять наличие колоды по имени перед вставкой
- Вызывается из `backend/db/seed.py` при старте в dev-режиме
- Данные карточек — Python-словари с DiagramSpec как вложенный dict

### Структура DiagramSpec для каждого типа

**term_definition / directional**:
```
{
  "wind": {"twd_deg": 0},
  "boat_a": {"hdg_deg": 45, "sails": {"main": {"boom_deg": 10, "state": "trimmed"}, "jib": {"state": "trimmed"}}},
  "expected_answer": {"type": "choose_option", "value": "бейдевинд"}
}
```

**right_of_way**:
```
{
  "wind": {"twd_deg": 0},
  "boat_a": {"position": {"x": 200, "y": 300}, "hdg_deg": 45, ...},
  "boat_b": {"position": {"x": 250, "y": 280}, "hdg_deg": 315, ...},
  "expected_answer": {"type": "select_boat", "value": "b"}
}
```

**manoeuvre**:
```
{
  "wind": {"twd_deg": 0},
  "boat_a": {"hdg_deg": 45, "sails": ...},
  "steps": [
    {"description": "Руль на ветер", "boat_a": {"hdg_deg": 0}},
    {"description": "Пересекаем левентик", "boat_a": {"hdg_deg": 315}},
    {"description": "Паруса на новый борт", "boat_a": {"hdg_deg": 315, "sails": ...}}
  ]
}
```

## 3. Тестирование

- Тест сида: загрузка в чистую БД, проверка количества колод и карточек
- Тест идемпотентности: повторная загрузка не дублирует данные
- Тест валидности DiagramSpec: каждая карточка рендерится в DiagramStage без ошибок (интеграционный тест)
