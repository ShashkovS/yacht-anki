## 3. Миграции базы данных

Создать новые yoyo-миграции для пяти таблиц:

### Таблица `decks` (глобальные колоды)
- Идентификатор, название, описание
- Флаг "встроенная" (builtin)
- Временные метки

### Таблица `cards` (глобальные карточки)
- Идентификатор, ссылка на колоду
- Тип шаблона (term_definition, directional, trim, manoeuvre, right_of_way)
- Текст вопроса, текст ответа
- JSON-спецификация диаграммы (diagram_spec)
- JSON-массив тегов
- Временные метки

### Таблица `card_states` (прогресс пользователя по карточке)
- Идентификатор, ссылка на пользователя и карточку (уникальная пара)
- JSON с FSRS-состоянием (difficulty, stability, retrievability)
- Фаза карточки (new, learning, review, relearning)
- Дата следующего показа (due)
- Дата последнего ревью
- Временные метки

### Таблица `review_log` (история ответов)
- Идентификатор, ссылка на пользователя и карточку
- Оценка (1–4: Again/Hard/Good/Easy)
- Запланированный интервал, фактический интервал
- Длительность ответа в мс
- Временная метка ревью

### Таблица `user_settings` (настройки повторения)
- Ссылка на пользователя (первичный ключ)
- Целевой retention (по умолчанию 0.9)
- Лимит новых карточек в день
- Лимит повторений в день (опционально)
- Временная метка обновления
