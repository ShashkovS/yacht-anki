# Фаза 08 — Полировка, PWA и оффлайн

## Цель
Довести приложение до production-ready состояния: надёжная работа оффлайн, мобильная оптимизация, импорт/экспорт прогресса, финальная полировка.

## Результат
Приложение устанавливается как PWA, работает без интернета, удобно на телефоне и готово к использованию на причале.

---

## 1. PWA и оффлайн

### Service Worker
- Настроить vite-plugin-pwa с Workbox:
  - Precache: все статические ассеты (JS, CSS, шрифты)
  - Runtime cache: API-ответы колод и карточек (они глобальные и редко меняются)
- Стратегия кеширования:
  - App shell — CacheFirst
  - API колод/карточек — StaleWhileRevalidate
  - API ревью — NetworkOnly (или очередь для оффлайн)

### Оффлайн-очередь ревью
- При отсутствии сети: сохранять результаты ревью в localStorage/IndexedDB
- При восстановлении сети: отправить накопленные результаты батчем
- Добавить бэкенд-эндпоинт `POST /review/submit-batch` для пакетной отправки
- UI-индикатор: "Оффлайн — результаты сохранены локально"

### PWA Manifest
- Название: "Яхтенный тренажёр"
- Иконки для разных размеров
- Display: standalone
- Theme color, background color
- Промпт установки на главном экране

## 2. Импорт/экспорт прогресса

### Экспорт
- Кнопка в настройках: "Экспортировать прогресс"
- Формат: JSON-файл с card_states и review_log пользователя
- Скачивается как файл

### Импорт
- Кнопка в настройках: "Импортировать прогресс"
- Предпросмотр: сколько записей будет импортировано
- Маппинг по card_id (карточки глобальные, id стабильные)
- Обработка конфликтов: новые данные перезаписывают старые

---

## 3. Мобильная оптимизация

### Touch-взаимодействие
- Проверить и доработать touch-события для react-konva (вращение яхты, тап)
- Крупные кнопки рейтинга (минимум 48px touch target)
- Свайп для перехода между карточками (опционально)

### Производительность
- Ленивая загрузка страниц (React.lazy)
- Оптимизация DiagramStage: не перерендеривать при неизменных пропсах
- Предзагрузка следующей карточки в очереди

### Визуальная полировка
- Анимации переходов между карточками
- Приятные цвета для галсов (зелёный/красный)
- Состояния загрузки (скелетоны)
- Пустые состояния ("Нет карточек к повторению")
