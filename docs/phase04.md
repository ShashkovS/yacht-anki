# Фаза 04 — Движок визуализации

## Цель
Создать набор React-компонентов на базе react-konva для рендеринга яхтенных ситуаций из DiagramSpec JSON. Движок должен быть чистым и stateless — рисует ровно то, что описано в спецификации.

## Результат
Компонент `DiagramStage` принимает DiagramSpec и рендерит полноценную 2D-сцену: яхты, паруса, ветер, знаки, зоны. Покрыт тестами.

---

## 1. Структура DiagramSpec

JSON-объект, хранящийся в поле `diagram_spec` карточки:

### Ветер
- Направление истинного ветра (градусы)
- Сила ветра (опционально, для отображения)

### Яхта (одна или две)
- Позиция на канвасе
- Курс (heading, градусы)
- Состояние парусов:
  - Грот: угол гика относительно ДП, состояние (добран/потравлен/полощет)
  - Стаксель: угол, состояние
  - Генакер: поставлен/убран, угол
- Цветовая маркировка галса (зелёный — правый, красный — левый)

### Знак (опционально)
- Позиция буя
- Радиус зоны (3 длины корпуса)

### Оверлеи (опционально)
- Подсветка наветренной/подветренной стороны
- Линии связанности
- Зона "сторониться"

---

## 2. Компоненты

### Корневой: `DiagramStage`
- Принимает DiagramSpec + размеры контейнера
- Адаптивный: масштабируется под экран (мобильный/десктоп)
- Konva `<Stage>` → `<Layer>` → дочерние компоненты

### `WindArrow`
- Стрелка (или несколько параллельных) показывающая направление истинного ветра
- Подпись "Ветер" и опционально скорость

### `BoatShape`
- Контур корпуса (вид сверху, упрощённый силуэт Fareast 28R)
- Мачта, гик, паруса как отдельные `<Line>` / `<Path>`
- Вращение всей группы по heading
- Цвет борта по галсу
- Опциональная подсветка ("эта яхта должна уступить")

### `SailGroup`
- Грот: треугольник от мачты по гику
- Стаксель: треугольник впереди мачты
- Генакер: "пузатая" форма от бушприта (если поставлен)
- Визуальное состояние: добранный парус плотный, потравленный — с зазором, полощущий — пунктир/волна

### `MarkBuoy`
- Круг-буй в заданной позиции
- Полупрозрачный круг зоны вокруг

### `OverlapIndicator`
- Линии траверза для визуализации связанности
- Подсветка наветренной/подветренной стороны

---

## 3. Вспомогательные функции

### `diagramHelpers.ts`
- Нормализация углов (0–359)
- Вычисление галса по курсу и направлению ветра
- Расчёт угла гика по курсу и углу ветра (упрощённая модель)
- Определение, полощет ли парус (курс слишком остро к ветру для текущей настройки)
- Расчёт вымпельного ветра (для отображения, не для физики)
