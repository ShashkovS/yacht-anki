# Фаза 02 — Backend API

## Цель
Реализовать серверные эндпоинты для чтения колод и карточек, управления ревью и пользовательскими настройками.

## Результат
Полный набор API-эндпоинтов, покрытых тестами. Фронтенд может получать колоды, карточки, очередь повторений и отправлять результаты.

---

## 1. Структура API

Все эндпоинты POST-based, возвращают `{ok: true, data: ...}` по существующему паттерну.

### Колоды и карточки (чтение)
- `POST /decks/list` — список всех колод (авторизация не обязательна, данные глобальные)
- `POST /decks/get` — одна колода по id с количеством карточек
- `POST /cards/list` — карточки колоды (по deck_id), с пагинацией

### Очередь повторений (требует авторизации)
- `POST /review/queue` — получить карточки, которые пора повторять
  - Учитывает daily_new_limit и daily_review_limit из user_settings
  - Возвращает карточки с card_state (или без него для новых)
  - Сортировка: сначала просроченные, потом новые
- `POST /review/submit` — сохранить результат ревью одной карточки
  - Принимает: card_id, rating (1–4), FSRS-состояние (рассчитанное на фронте), elapsed_ms
  - Создаёт/обновляет card_state
  - Добавляет запись в review_log
- `POST /review/summary` — сводка для дашборда
  - Количество карточек к повторению сегодня
  - Количество новых карточек на сегодня
  - Общая статистика (всего изучено, streak и т.п.)

### Настройки пользователя
- `POST /settings/get` — получить настройки (или дефолтные, если нет записи)
- `POST /settings/save` — сохранить настройки

## 2. Организация кода

По существующему паттерну проекта:

### Backend DB-модули
- `backend/db/decks.py` — SQL-запросы для колод
- `backend/db/cards.py` — SQL-запросы для карточек
- `backend/db/card_states.py` — SQL-запросы для состояний карточек
- `backend/db/review_log.py` — SQL-запросы для лога ревью
- `backend/db/user_settings.py` — SQL-запросы для настроек

### Backend HTTP-маршруты
- `backend/http/deck_routes.py` — эндпоинты колод и карточек
- `backend/http/review_routes.py` — эндпоинты ревью
- `backend/http/settings_routes.py` — эндпоинты настроек

Регистрация маршрутов — в `backend/http/routes.py` через отдельные setup-функции.

## 3. Тестирование

### Юнит-тесты бэкенда
- `backend/tests/test_decks.py` — список колод, получение колоды
- `backend/tests/test_cards.py` — список карточек колоды
- `backend/tests/test_review.py` — очередь, сабмит, сводка
- `backend/tests/test_settings.py` — получение/сохранение настроек

Тесты используют существующие фикстуры (тестовая БД, авторизованный клиент).
